# useful-appcoll

Daily automated generation of a prioritized, task-enriched patent docket report written as a Markdown file to an Obsidian Vault, driven by AppColl CSV exports.

---

## How It Works

1. A macOS launchd job runs `run_docket.sh` every weekday at 9 AM Eastern.
2. The script logs into AppColl via headless browser and downloads the latest Tasks CSV to `data/appcoll_exports/`.
3. Each docket entry is scored by how many days remain until its filing deadline.
4. Entries above a configurable threshold have tasks generated by a YAML rules engine.
5. Tasks get subpriority labels (A.1, A.2 ...) and target dates; overdue tasks appear in ALL CAPS.
6. A Markdown report is written directly to your Obsidian Vault at `Docket/Daily/Docket_YYYY-MM-DD.md`.

## Setup

### Prerequisites

- Python 3.13+ (pyenv recommended: `pyenv install 3.13.12`)
- An Obsidian Vault at `~/Documents/ObsidianVault` (or configure via `OBSIDIAN_VAULT_PATH`)

### 1. Create virtual environment and install dependencies

```bash
python3.13 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
playwright install chromium
```

### 2. Configure credentials

```bash
cp .env.example .env
chmod 600 .env
# Edit .env and fill in APPCOLL_EMAIL and APPCOLL_PASSWORD
```

### 3. Configure output path (optional)

Edit `config/globals.py` or add to `.env`:

```bash
OBSIDIAN_VAULT_PATH=/path/to/your/vault
```

Other tunable constants in `config/globals.py`:
- `TASK_GENERATION_THRESHOLD` — minimum score to generate tasks (default: 30)
- `OBSIDIAN_DOCKET_SUBFOLDER` — subfolder within the vault (default: `Docket/Daily`)

### 4. Register the launchd job

```bash
launchctl load ~/Library/LaunchAgents/com.useful_appcoll.docket.plist
```

The job runs every weekday at 9 AM local time. Output is appended to `logs/docket.log`.

## Running Manually

```bash
# Run via launchctl (output goes to logs/docket.log)
launchctl start com.useful_appcoll.docket

# Run the script directly (output to terminal — useful for debugging)

```

## Customizing Rules

Rules live in `config/rules.yaml`. Each rule specifies conditions (field matches) and tasks (name + days-before-deadline). No code changes needed — edit the YAML and the next run picks it up.

See the comments at the top of `config/rules.yaml` for the operator reference and add/edit/delete instructions.

## Customizing Task Help Fields

`config/task_help.yaml` maps each task's `help_key` to a list of AppColl fields shown inline with that task in the report.

### _Note on Column Headers_

Column headers from the CSV file are normalized and mapped to python names. The yaml files must refer to the python names. The column mapping is shown in `scripts/csv_loader.py` in the variable `COLUMN_MAP`.
