# useful-appcoll

Daily automated generation of a prioritized, task-enriched patent docket report written as a Markdown file to an Obsidian Vault, driven by AppColl CSV exports.

---

## How It Works

1. A CSV is exported from AppColl (or downloaded automatically) and placed in `data/appcoll_exports/`.
2. Each docket entry is scored by how many days remain until its filing deadline.
3. Entries above a configurable threshold have tasks generated by a YAML rules engine.
4. Tasks get subpriority labels (A.1, A.2 ...) and target dates; overdue tasks appear in ALL CAPS.
5. A Markdown report is written to `vault_output/Docket/Daily/Docket_YYYY-MM-DD.md`.
6. GitHub Actions runs this daily and commits the output; Obsidian Git pulls it to your vault.

## Setup

### Prerequisites

- Python 3.13+ (pyenv recommended: `pyenv install 3.13.12`)
- An Obsidian Vault with the Obsidian Git plugin
- A private GitHub repository for this project

### Create virtual environment and install dependencies

```bash
# Create venv with Python 3.13
python3.13 -m venv .venv
source .venv/bin/activate

pip install -r requirements.txt
```

### Configure

Edit `config/globals.py` to set:
- `OBSIDIAN_VAULT_PATH` — path to your Obsidian Vault (overridable via env var)
- `TASK_GENERATION_THRESHOLD` — minimum score to generate tasks (default: 30)
- `OBSIDIAN_DOCKET_SUBFOLDER` — subfolder within the vault (default: `Docket/Daily`)

### Run locally

```bash
# Activate venv first (if not already active)
source .venv/bin/activate

# Place an AppColl CSV export in data/appcoll_exports/, then:
python scripts/main.py
```

The report appears at `~/Documents/ObsidianVault/Docket/Daily/Docket_YYYY-MM-DD.md`.

### Run tests

```bash
source .venv/bin/activate
python -m pytest tests/ -v
```

## GitHub Actions (CI)

The workflow runs every weekday at 6 AM Eastern.

**Required GitHub repository secret:**

| Secret | Description |
|---|---|
| `APPCOLL_CSV_URL` | Direct URL to download the AppColl CSV export |

The workflow downloads the CSV, generates the report, and commits the `.md` file back to this repo using the built-in `GITHUB_TOKEN` (no additional secrets needed for the commit step).

**Obsidian sync:** Install the [Obsidian Git](https://github.com/denolehov/obsidian-git) plugin and configure it to pull from this repository. Point the vault root at the `vault_output/` directory (or adjust `OBSIDIAN_DOCKET_SUBFOLDER` in `globals.py`).

## Customizing Rules

Rules live in `config/rules.yaml`. Each rule specifies conditions (field matches) and tasks (name + days-before-deadline). No code changes needed — edit the YAML and the next run picks it up.

See the comments at the top of `config/rules.yaml` for the operator reference and add/edit/delete instructions.

## Customizing Task Help Fields

`config/task_help.yaml` maps each task's `help_key` to a list of AppColl fields shown inline with that task in the report.

---

## Security Note

Never commit raw AppColl CSV files to a public repository. The `.gitignore` excludes `data/appcoll_exports/*.csv`. Use a **private** GitHub repository.
